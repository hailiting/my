架构设计 · AI + Python + 跨端

工程架构

本作品集采用双轨交付：PageDrop 静态站保证大陆访问速度； Vercel Python Serverless 承载 DeepSeek 代理与访问统计；Flutter 工程（portfolio_app）展示跨端架构能力。 AI 研发侧融入 Claude Code、Cursor Skills 与 MCP，Agent 编排向 LangChain / LangGraph 演进。

01 AI 研发体系 · Claude Code + Skills + MCP

人机协同：约束在架构层，生成在实现层

将大模型与 IDE Agent 纳入可验收的工程流程，而非「无边界自动生成」。

  • Claude Code — 终端/IDE 内 Agent 式多文件迭代，适合复杂重构与跨目录任务
  • Cursor — 主力 IDE；Rules / Composer Agent 驱动需求→代码→测试闭环
  • Trae · SOLO Coder — 从 0 到 1 的完整页面/模块交付
  • Skills — 可复用技能包（如部署、PR、Canvas、MCP 集成），把团队规范固化为 Agent 可执行步骤
  • MCP（Model Context Protocol） — 为 Agent 挂载 GitHub、文档、数据库等外部上下文，减少幻觉、提高可验证性
  • 人机分工：架构决策、密钥边界、业务取舍 — 人；样板代码、文档草稿、测试骨架 — AI
需求 → Skills 选路 → MCP 拉上下文 → Claude Code / Cursor 生成
     → flutter analyze / pytest → 人 Review → 合并

02 Python 后端 · Vercel Serverless

无状态函数 + 共享 lib，API Key 仅存服务端

正式站静态资源与计算分离：前端跨域调用 Vercel API（已配置 CORS *）。

  • api/ai-about.py — POST 代理 DeepSeek，body { "keyword": "区块链" }
  • api/visit.py — POST 记录访问 / GET 聚合统计（Upstash Redis）
  • lib/deepseek.py — Prompt 背景、token 上限、503 本地回退契约
  • lib/analytics.py — 会话去重、PV / UV 计数
  • lib/http_util.py — JSON 读写、CORS、OPTIONS 预检
  • 部署：vercel.json 指定 legacy/static-site 输出 + Python 3.12 运行时
my/
├── api/           ← Vercel 入口（handler 类）
├── lib/           ← 业务逻辑（可单测）
├── legacy/static-site/   ← PageDrop / Vercel 静态
└── vercel.json

03 LangChain · LangGraph · Agent 编排

从「单次补全」到「有状态工作流」

本站 AI 介绍当前为轻量代理链（System Prompt → DeepSeek Chat）。工程演进方向是用 LangChain / LangGraph 组织可观测、可分支的 Agent 流水线。

  • LangChain — LCEL 链式组合：检索履历片段 → 约束 Prompt → 调用模型 → 结构化输出
  • LangGraph — 有向图状态机：关键词分类 → 选技能模板 → 生成 → 人工审核节点（可选）
  • 典型节点：意图识别、事实校验（仅允许 Profile 内经历）、token 预算、失败回退本地模板
  • 与 MCP 结合：Graph 节点调用 MCP 工具拉取仓库/文档，再写入生成上下文
  • 可扩展：多轮对话记忆、A/B Prompt、访问统计与生成质量联动分析
[访客 keyword]
    → Router (LangGraph)
    → Retrieve(profile) ──MCP──► repo / resume
    → Generate (DeepSeek via LangChain)
    → Fallback (local template) if 503 / timeout
