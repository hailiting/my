架构设计 · 现代化工程体系

工程架构

状态可预测、路由可声明、边界可划分——本作品集 Flutter 工程按此标准搭建，代码结构即能力证明。

01 状态管理 · Riverpod

可预测的单向数据流，UI 只订阅状态

使用 flutter_riverpod 管理主题、路由与技能雷达权重。Provider 让依赖显式、可测试、可组合，避免 setState 散落各处。

  • StateProvider：雷达权重等轻量 UI 状态
  • Provider：GoRouter 等依赖注入
  • ref.watch / ref.read 区分重建与一次性读取
  • 状态定义靠近 Feature，Widget 只负责展示
// features/skills/skills_page.dart
final radarWeightsProvider = StateProvider<List<double>>((ref) => ...);
final weights = ref.watch(radarWeightsProvider);

02 路由 · go_router

声明式路由表 + 深度链接

统一管理站内路径，Shell 布局常驻导航，路径参数同步浏览器地址栏，便于分享。

  • ShellRoute：AppShell 包裹各业务页
  • /skill/:slug — 技能证据深度链接
  • /architecture — 本架构说明页
  • navigatorKey + context.go 可编程导航
GoRoute(path: '/skill/:slug', builder: ... SkillDetailPage),
GoRoute(path: '/architecture', builder: ... ArchitecturePage),

03 代码架构 · Feature-first

分层清晰，代码即作品

Feature-first + Clean 思想：core 放领域数据，features 垂直切业务，widgets 放可复用 UI，router 独立配置。

  • core/ — 常量、主题、通用能力
  • features/ — home / skills / architecture / projects …
  • widgets/ — 词云、雷达图、项目卡片
  • router/ — go_router 单一入口
portfolio_app/lib/
├── core/
├── features/architecture/  ← 本页
├── widgets/
└── router/app_router.dart
